# Занятие 10: Мастерство — методы саморазвития и культура работы

> **Фокус занятия:** Ступень 4 «Дисциплинированный». Не старт, а сборка.
> Всё, что ниже, уже должно быть в практике — здесь мы делаем одну карту.

---

## Ключевое различение: два вида методов

В программе ЛР ты осваиваешь **два принципиально разных вида методов**. Их часто путают, потому что они работают рядом. Но путать их — значит чинить не тот слой.

### Методы саморазвития

Это практики, которые развивают **тебя как человека**.
Объект изменения — твои способности, привычки, стиль жизни.

Читаешь медленно → развиваешь способность осваивать сложное.
Проговариваешь вслух → развиваешь мышление.
Ведёшь учёт времени → развиваешь осознанность к ресурсу.

Вопрос, на который отвечают: **«Что я делаю, чтобы расти как человек?»**

### 14 элементов культуры работы

Это правила и паттерны того, **как устроена работа**.
Объект изменения — система работы, а не человек.

ОРЗ, каскад планирования, АрхГейт, стоп-краны — не про развитие тебя,
а про то, как правильно организовать рабочий процесс.

Вопрос, на который отвечают: **«Как я работаю внутри рабочей среды?»**

### Аналогия

Если ты футболист:
- Методы саморазвития = тренировки (бег, физподготовка, техника удара). Развивают игрока.
- 14 элементов = правила игры и тактика команды. Определяют, как игрок действует на поле.

Можно быть отлично натренированным, но играть хаотично без тактики.
И наоборот — знать все правила, но быть слабым физически.

### Почему они путаются

Некоторые темы совпадают: учёт времени есть и там, и там. Планирование — тоже.
Различие не в теме, а в **уровне анализа**:

| Уровень | Пример |
|---------|--------|
| **Метод M001** «учёт времени» | Личная практика: ты осваиваешь способность видеть реальность своего времени |
| **Элемент 3** «ритуал учёта» | Правило рабочего процесса: как вести учёт в системе — по РП, с бюджетом, с привязкой к результату |

Первое про тебя. Второе про систему.

## Блок А. 8 методов саморазвития

Методы — это операционализированные практики с понятным входом, действием и измеримым выходом. Без них развитие случайное. Их ровно восемь.

| № | Метод | Что развивает | Индикатор освоения |
|---|-------|--------------|-------------------|
| **M001** | Учёт времени | Видение реального распределения ресурса | WakaTime / трекер показывает факт, а не предположение |
| **M002** | Метод ученика | Регулярность освоения: слоты → экзокортекс → рефлексия → контракт | Недельный контракт есть, выполнен — проверяемо по артефакту |
| **M003** | Медленное чтение | Способность осваивать сложный текст и применять в работе | Есть тезис из чтения, изменивший решение в текущей задаче |
| **M004** | Мышление в письме | Способность порождать мысль через текст, а не записывать готовую | Fleeting notes есть. Хотя бы одна стала черновиком на этой неделе |
| **M005** | Мышление проговариванием | Способность обнаруживать пустоты через объяснение | На этой неделе поймал хотя бы одну подмену понятий в своём же рассуждении |
| **M006** | Организация досуга | Восстановление и баланс как управляемый процесс, а не случайный | Досуг запланирован, а не «что осталось после работы» |
| **M007** | Формирование окружения | Осознанный выбор среды: усилители / тормоза / зеркала | За последний месяц было хотя бы одно намеренное изменение в окружении |
| **M008** | Стратегирование | Связь «зачем» с «что делаю сейчас» на разных горизонтах | Ключевой вопрос «К какому результату ведёт то, что я делаю сейчас?» — есть ответ |

### Три слоя освоения

```
СЛОЙ 1 — Фундамент (ступени 1–2)
  M001 Учёт времени
  M002 Метод ученика (минимальный цикл)
  M006 Организация досуга

СЛОЙ 2 — Основной (ступени 2–3)
  M003 Медленное чтение
  M004 Мышление в письме
  M005 Мышление проговариванием

СЛОЙ 3 — Продвинутый (ступени 3–4)
  M007 Формирование окружения
  M008 Стратегирование
```

**Принцип:** один метод идёт на глубину за раз. Параллельный запуск нескольких — не тянет ресурс.
**Bottleneck** определяет приоритет. Не «освою всё сразу», а «что сейчас тормозит».

## Блок Б. 14 элементов культуры работы

Культура работы — это не дополнительная нагрузка. Это **способ делать** уже существующую работу.

### Протоколы (формализованные последовательности)

**1. ОРЗ — Открытие, Работа, Закрытие**

Три стадии для любой работы — от 15-минутной сессии до месяца.

- **Открытие:** роль, метод, критерии готовности. Без этого непонятно, когда закончить
- **Работа:** фиксировать знания сразу, не «потом»
- **Закрытие:** документировать результат и выводы. Без закрытия знание теряется

> Пропустил Открытие = незапланированная работа.
> Пропустил Закрытие = незафиксированный результат. Обе ошибки одинаково дороги.

---

**2. Каскад планирования**

Год → Месяц → Неделя → День. Каждый уровень сужает фокус.

Частая ошибка: планировать день без недельного контекста. Тогда «срочное» выталкивает «важное» — и это не дисциплинарная проблема, а архитектурная.

---

**3. Ритуал учёта времени**

Учёт по рабочим продуктам, а не по задачам. Каждый РП: номер + бюджет часов + приоритет + связь с месячным результатом. Перед сессией: роль и метод названы явно.

---

### Форматы (стандарты оформления)

**4. Организация знаний — иерархия истины**

Три уровня доверия:
```
Принципы (FPF, ZP)     ← самый высокий вес
  Pack (доменное)       ← истина в предметной области
    DS (код, документы) ← производное, меняется следом за Pack
```
При противоречии: производное уступает Pack, Pack уступает принципам.

---

**5. ВДВ — Вход, Действие, Выход**

Три аспекта любой системы. Тест на полноту: у каждого процесса есть явный ответ:
- **Вход:** откуда данные? кто пишет?
- **Действие:** какой метод? какая роль?
- **Выход:** кому результат? где хранится?

Если нет ответа на один вопрос — система неполная.

---

### Навыки (нарабатываемые способности)

**6. Захват знания — Capture-to-Pack**

На каждом рубеже: «Есть ли знание, которое стоит записать?»

Маршрутизация: правило → CLAUDE.md, доменное → Pack, урок → memory/, реализационное → DS docs/.
Без Capture знание растворяется. С Capture — накапливается в системе.

---

**7. АрхГейт**

7 характеристик для архитектурного решения: эволюционируемость, масштабируемость, обучаемость, генеративность, скорость, современность, безопасность.

Правило: архитектурное решение → сначала АрхГейт → потом реализация. Прыжок сразу в реализацию — одна из самых дорогих ошибок.

---

**8. Стоп-краны — Pre-action Gates**

Автоматические проверки перед действием:
```
Задание → задание в плане?
Сценарий → обещание описано?
Архитектура → АрхГейт пройден?
РП → роль и метод определены?
```

---

**9. Осторожный мультитаскинг**

Параллельная работа в нескольких окнах ИИ с разными РП без смешивания контекстов.
Мультипликатор: 1,5–2,5×. Больше 3–4 параллельных сессий — координационные издержки перекрывают выигрыш.

---

**10. Мультиагентность**

Один агент усиливает. Два агента с независимым контекстом — система контроля качества.
Результат одной модели проверяется другой с чистым контекстом. Даже слабая модель регулярно ловит ошибки сильной — потому что контекст не замылен.

---

**11. Экзоскелетный режим**

- **Экзоскелет** = усиливает функцию, человек в петле
- **Протез** = замещает функцию, человек выпадает

Тест: «Если завтра инструменты исчезнут — я продолжу работать, просто медленнее?»
Да → экзоскелет. Нет → протез.

---

**12. Самоисправление**

Расхождение с реальностью → немедленное исправление: что, где, как изменить.
Не «разберёмся потом». Без Capture самоисправление не работает — исправление теряется вместе со знанием.

---

**13. Стоп-лист для рабочих продуктов**

Четыре критерия прекращения РП:
1. Бюджет исчерпан на 150%+ без значимого прогресса
2. Блокер стоит дольше двух недель
3. Гипотеза опровергнута
4. Приоритет упал — другие РП важнее

Продолжать РП из инерции — один из главных пожирателей ресурса.

---

**14. Эволюция системы**

При закрытии недели — три вопроса:
```
1. Что вынес? (знание, которое осталось)
2. Какое правило мешало?
3. Какое правило обходил?
```
Система меняется на основе реальных столкновений с реальностью, не из теории.

---

## Как два блока работают вместе

Методы и культура работы дополняют друг друга. Метод определяет, что развивать. Культура — как делать работу внутри которой идёт развитие.

| Метод саморазвития | → | Элемент культуры, который его поддерживает |
|--------------------|---|-------------------------------------------|
| M001 Учёт времени | → | Ритуал учёта (эл. 3) |
| M003 Медленное чтение | → | Захват знания (эл. 6) |
| M004 Мышление в письме | → | ОРЗ + ВДВ (эл. 1, 5) |
| M008 Стратегирование | → | Каскад планирования (эл. 2) |
| Любой метод | → | Эволюция системы (эл. 14) — рефлексия меняет и методы |

## Упражнение после занятия

**Шаг 1. Диагностика методов (15 мин)**

Для каждого из 8 методов:
```
✅ В практике — есть артефакт этой недели
⚠️ Знаю — нет регулярного артефакта
❌ Не начинал
```

**Шаг 2. Диагностика культуры (15 мин)**

Для каждого из 14 элементов:
```
✅ Работает автоматически — не требует сознательного усилия
⚠️ Применяю — но не всегда и не автоматически
❌ Не применяю
```

**Шаг 3. Один следующий шаг (5 мин)**

> «На этой неделе я встрою [метод или элемент] в [конкретную существующую работу]
> и проверю по [конкретному индикатору].»

## Итог занятия

Мастерство — это автоматизм применения без сознательного усилия.

Пока требуется усилие — это внимание, не мастерство.
Когда усилие не нужно — высвобождается внимание для следующего.

Карта из этого занятия — не чеклист для выполнения.
Это **зеркало**: что уже стало системой, что ещё требует внимания.
